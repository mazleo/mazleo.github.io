To explain a bit about the details of the implementation, the technique used here uses the Perlin Noise. Perlin noise is essentially noise that consists of gradual changes in values.

For example, here is an image of noise:

If we think of these colors as values, the changes in values are non-gradual. Perlin noise, on the other hand, consists of gradual changes in values:

We see that there is a gradual change from white to black (in other words, shades of gray). This is why Perlin Noise is commonly used for procedural terrain. Instead of treating these as colors, these can be treated as height values so that darker blacks are heigher in altitude for example.

The usage of Perlin noise differs for each project. In this iteration, multiple perlin noise models are used. Here, there are separate models for each biome: alpine, aquatic, and grasslands biome models. Each one have their own parameters as to how tight the wavelengths are and how high the amplitudes of the waves are. For example, grasslands will not have much waves and hence have lower frequency or have looser waves, but also has less amplitudes so the waves aren't very high. On the other hand, the alpine biome has higher frequency and amplitude for its waves. To combine these together, there is another model which determines which biomes are where. In other words, certain values at certain points map to a specific biome. ie. maybe values from 0.0 to 0.3 is the aquatic biome, 0.3 to 0.6 for grasslands, and 0.6 to 1.0 for alpine. Since the locations of the biomes are now determines, the model specific to the biome is used in that location.
